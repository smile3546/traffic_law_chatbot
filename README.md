# 專案名稱：traffic_law_chatbot

本專案是一款專為中華民國交通法規與車禍理賠實務設計的智慧諮詢專家系統。系統基於 Gemini 3.1 Flash-Lite 核心大模型，透過動態脈絡增強管線，深度整合《道路交通管理處罰條例》、《道路交通安全規則》以及產險公會最新《汽車道路交通事故分擔罪責劃分標準》，實現具備高精確度與可解釋性的法規檢索與複合多車肇責權重推理。

## 核心技術亮點

* 動態脈絡注入：繞過傳統向量資料庫的檢索流失風險，充分利用 Gemini 3.1 百萬級別的長文本上下文視窗，將逾 10 萬字的非結構化交通法規與理賠準則精煉為結構化特徵，達成全文本 100% 的注意力覆蓋，徹底消除中段資訊遺失現象。
* 多維度特徵權重計算：系統不僅能進行單一條文對齊，更內建產險公會「肇責增減扣抵演算法邏輯」。當面對複合型違規情境（如支道未讓幹道 + 主因車超速 + 幹道車未減速）時，模型能精確調用 70% / 30% 主次因權重，並動態進行 ±5% 權重微調計算，給出絕對精確的裁罰與分攤百分比。
* 生產級語意去噪：後端管線整合了正則表達式濾波器，在模型文本生成階段即時清洗中繼資料與 Token 殘留雜訊（如特定 In-context Learning 衍生之代碼標籤），確保交付給前端 UI 的回應皆為純淨、嚴謹的法律白話文。
* 極致的端到端延遲優化：選用輕量化、低延遲的 Gemini 3.1 Flash-Lite，使高達 13 萬 Token 的 Prompt Processing（首字輸出延遲）控制在 1 ~ 1.5 秒內，提供流暢的即時對話體驗。

## 系統架構

使用者當前輸入情境描述，透過 Streamlit 建立的前端網頁或行動端介面發送請求。後端管線將輸入與加載的 10 萬字多源交通知識庫文本進行結構化動態提示詞拼接。隨後送入 Gemini 3.1 Flash-Lite 核心進行推理（溫度設定為 0.3），生成之文本經過正則表達式組成的語意去噪與數據清洗層後，最終輸出乾淨、精確的法規條文與肇責百分比給使用者。

## 知識庫架構範疇

本系統之知識工程融合了以下多源權威數據：
1. 《道路交通管理處罰條例》：罰鍰金額計算、違規記點與行政裁罰依據。
2. 《道路交通安全規則》：轉彎車讓直行車、少線道讓多線道等核心路權與行車行為準則。
3. 產險公會《汽車道路交通事故分擔罪責劃分標準》：
    * 同向與對向事故判定（例如：跨越雙黃線、單行道逆向、起駛、開車門衝突）。
    * 特殊地形會車路權（例如：峻狹坡道下坡讓上坡、山路靠山壁車讓外緣車原則）。
    * 多車連環事故（追撞與推撞）責任拆分矩陣。
    * 私人場域（例如：大賣場地下停車場、貨櫃場）之法律準則適用擴展。
    * 路權受損與號誌各執一詞之 50% 互賠實務。

## 開發環境與核心依賴

* 開發語言：Python 3.10+
* 核心模型 API：Google Generative AI SDK (gemini-3.1-flash-lite)
* 網頁框架：Streamlit
* 環境變數管理：python-dotenv
* 文本處理：Regular Expressions (re)

## 快速啟動與跨裝置部署

1. 複製專案與安裝依賴：
    ```bash
    git clone [https://github.com/your-username/traffic_law_chatbot.git](https://github.com/your-username/traffic_law_chatbot.git)
    cd traffic_law_chatbot
    pip install -r requirements.txt
    ```

2. 環境變數配置：
    於專案根目錄建立 `.env` 檔案並填入您的 Gemini API Key：
    ```text
    GEMINI_API_KEY=your_actual_api_key_here
    ```

3. 本地啟動：
    ```bash
    streamlit run app.py
    ```


## 免責聲明

本系統輸出之所有肇責比例、裁罰金額與法規判定，均基於既有知識庫文本進行語意推理與量化模擬，僅供保險理賠實務與交通事故諮詢之邏輯參考，不具備最終法律訴訟之裁判效力。實際個案仍須以法院判決書及車輛行車事故鑑定委員會之意見書為準。
